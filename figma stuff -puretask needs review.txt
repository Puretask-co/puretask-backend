Step‑by‑Step Guide to Building the Design System in Figma
This guide explains how to set up a complete design system and create key interface screens in Figma. It assumes you are new to Figma and walks through fundamental concepts like frames, grids, auto‑layout, components, variants, typography and color styles. The approach follows best practices described in accessible tutorials on auto‑layout
 and design‑system organisation
.

1. Global Design System
1.1 Frame sizes
Open Figma. Press F (Frame tool). In the right panel, choose a desktop size and create frames sized:

Desktop App: 1440 × 1024

Tablet: 1024 × 900 (optional)

Mobile: 390 × 844 (optional)

Name the frames clearly, e.g., AppShell/Desktop. You can duplicate these frames for other sizes later.

1.2 Layout Grid (12‑column)
Select the desktop frame.

In the right‑hand properties panel, find Layout grid and click + to add a grid.

Change the grid type to Columns and set:

Count: 12 columns

Margin: 80 px (left and right)

Gutter: 24 px between columns

Use an 8‑point spacing system for all padding and spacing values. A 4‑point grid helps maintain rhythm and responsiveness
.

You can use similar grids on other frames (e.g., 12 columns for desktop, fewer columns for smaller breakpoints) by copying the grid and adjusting the column count and margins
.

1.3 Spacing system
Follow an 8‑point scale, which ensures that all paddings and margins are multiples of 4 or 8. Suggested values: 4, 8, 12, 16, 24, 32, 48, 64. Use these numbers consistently for padding, gaps, border radius and shadows.

1.4 Color tokens
Open the Assets panel and switch to Library > Local styles.

Click + to add Color Styles. Name them using a clear hierarchy, for example:

Group	Tokens
Primary	Primary/600, Primary/500
Success	Success/500
Warning	Warning/500
Error	Error/500
Neutral	Neutral/900, Neutral/700, Neutral/500, Neutral/300, Neutral/100
Surface	Surface/Card (for cards), Surface/App (for page backgrounds)
Apply these color styles to shapes and components so that any changes propagate automatically. Figma makes it easy to define a color palette and reuse styles across the system
.

1.5 Typography styles
Figma allows you to create text styles for headings, body and captions. To set them up:

Use the Text tool (T) to create text elements for each style (H1, H2, H3, Body, Small, Caption).

In the right panel, define font size and weight:

H1 – 32 px / 600 weight

H2 – 24 px / 600

H3 – 18 px / 600

Body – 16 px / 400

Small – 14 px / 400

Caption – 12 px / 400

Click the four‑dots icon next to Text in the properties panel to create a new Text Style. Give each style a descriptive name (e.g., Heading/H1), so you can apply it easily later
.

These styles provide a hierarchy that helps users scan the content easily
.

2. Fundamentals: Frames, Auto‑layout & Components
2.1 Using Auto‑layout
Auto‑layout is Figma’s equivalent of flexbox. It lets you define how child elements are spaced and aligned. According to a beginner tutorial
, auto‑layout only works on frames or components and not on groups. You can add auto‑layout by:

Selecting one or more layers.

Pressing Shift + A (shortcut).

Clicking + Auto layout in the right panel.

Right‑clicking and choosing Add auto layout
.

Once applied, auto‑layout can be set to Horizontal or Vertical. You can rearrange elements by dragging them or using the arrow keys, and switch between horizontal and vertical flows using the orientation controls
.

2.2 Creating Components and Variants
Components turn your designs into reusable building blocks. A design system relies on components and their variants to maintain consistency
. To create a component:

Select the elements (e.g., a button rectangle and text).

Apply auto‑layout if the component needs flexible padding (e.g., a button should hug its text content).

With the group selected, press Ctrl + Alt + K (⌘ + Option + K on Mac) to create a component.

Name the component logically (e.g., Button/Primary).

To create variants, select the component and click + Variant in the right panel. Variants allow multiple states (e.g., primary, secondary, danger, disabled) to be stored in one master component. You can create variant properties such as State and assign values like Default, Hover, or Active to each variant. Grouping states as variants keeps your library organised and easy to use
.

2.3 Applying Component Properties
Figma’s component properties enable you to define which parts of a component can change. You can add Boolean properties (e.g., show/hide an icon), Instance swap properties (e.g., replace an icon), and Text properties (e.g., change button label)
. This reduces the number of required variants and makes the system easier to scale.

3. Building the App Shell
The App Shell provides a reusable layout containing a sidebar and main content area. Follow these steps:

Create the Frame: Press F, choose 1440 × 1024 desktop size and name it AppShell/Desktop.

Enable Auto‑layout: Select the frame and press Shift + A to convert it to auto‑layout. Set the direction to Horizontal, so children are arranged left‑to‑right.

Create the Sidebar:

Draw a rectangle (240 px wide, height set to Fill container). Name it Sidebar.

With the rectangle selected, press Shift + A to apply a Vertical auto‑layout.

Add padding of 24 px and set spacing between items to 16 px.

Insert the logo (a 32 px‑high image or placeholder) at the top.

For each navigation item, create a row (40 px high) with a 20 px icon and label. Use auto‑layout horizontally to align icons and labels and apply consistent padding. Turn the row into a component (NavItem) with default, hover, and active variants.

Create the Main Content Area:

Draw another rectangle to the right of the sidebar and set its width to Fill container and height to Fill container.

Apply vertical auto‑layout with padding 48 and spacing 32.

Set the fill color to Surface/App and the frame’s corner radius if needed (e.g., 0 for full‑screen). All pages (progress hub, goals list, etc.) will live inside this area.

11. Creating Component Variants

To build each variant, duplicate the base component and modify attributes such as colour or text. Use variant properties like type, status or state to switch between variants. Using Figma’s Variants feature groups them into a single component, making it easy to swap between states

12. Responsive Rules
Responsive design ensures your interface adapts to smaller viewports:

Sidebar collapses: At narrow widths, change the Sidebar component variant to an icon‑only style (e.g., hide labels and show only icons). You can implement this with a Boolean property to toggle label visibility.

Goal cards: On mobile, stack cards in a single column. Use auto‑layout with Wrap enabled
 to allow items to flow onto new lines.

Level summary: Stack the left and right content vertically when the container width becomes narrow. This can be achieved by creating a responsive variant or by switching auto‑layout orientation based on the parent frame width.

Reward cards: Replace the horizontal scroll container with a vertical list on mobile.

13. Structuring Your Design System Page
To keep your components organised:

Create a new page in Figma called “Design System”.

Add headings (e.g., Buttons, Cards, Typography, Colors, Badges, GoalCard, RewardCard, NavItem, MetricCard).

Place your components under each section. Use auto‑layout to keep them aligned.

Document usage guidelines within Figma notes or use a tool like Notion to write detailed documentation. Make sure to explain the purpose of each component, its variants, and when to use each state.

14. Prototyping
To make your designs interactive:

Select a component (e.g., a button) and switch to the Prototype tab in the right panel.

Drag the blue node from the selected element to the destination frame (e.g., from a GoalCard button to a Goal details page).

Choose Interaction (e.g., On click), Action (Navigate to), and Animation (e.g., Smart animate).

Repeat this for all interactions you want to showcase.

You can use similar grids on other frames (e.g., 12 columns for desktop, fewer columns for smaller breakpoints) by copying the grid and adjusting the column count and margins
.

1.3 Spacing system
Follow an 8‑point scale, which ensures that all paddings and margins are multiples of 4 or 8. Suggested values: 4, 8, 12, 16, 24, 32, 48, 64. Use these numbers consistently for padding, gaps, border radius and shadows.

1.5 Typography styles

Use the Text tool (T) to create text elements for each style (H1, H2, H3, Body, Small, Caption).

These styles provide a hierarchy that helps users scan the content easily
.

2. Fundamentals: Frames, Auto‑layout & Components
2.1 Using Auto‑layout
Auto‑layout is Figma’s equivalent of flexbox. It lets you define how child elements are spaced and aligned. According to a beginner tutorial
, auto‑layout only works on frames or components and not on groups. You can add auto‑layout by:

Selecting one or more layers.

Pressing Shift + A (shortcut).

Clicking + Auto layout in the right panel.

Right‑clicking and choosing Add auto layout
.

Once applied, auto‑layout can be set to Horizontal or Vertical. You can rearrange elements by dragging them or using the arrow keys, and switch between horizontal and vertical flows using the orientation controls
.

2.2 Creating Components and Variants
Components turn your designs into reusable building blocks. A design system relies on components and their variants to maintain consistency
. To create a component:

Select the elements (e.g., a button rectangle and text).

Apply auto‑layout if the component needs flexible padding (e.g., a button should hug its text content).

With the group selected, press Ctrl + Alt + K (⌘ + Option + K on Mac) to create a component.

Name the component logically (e.g., Button/Primary).

To create variants, select the component and click + Variant in the right panel. Variants allow multiple states (e.g., primary, secondary, danger, disabled) to be stored in one master component. You can create variant properties such as State and assign values like Default, Hover, or Active to each variant. Grouping states as variants keeps your library organised and easy to use
.

2.3 Applying Component Properties
Figma’s component properties enable you to define which parts of a component can change. You can add Boolean properties (e.g., show/hide an icon), Instance swap properties (e.g., replace an icon), and Text properties (e.g., change button label)
. This reduces the number of required variants and makes the system easier to scale.

3. Building the App Shell
The App Shell provides a reusable layout containing a sidebar and main content area. Follow these steps:

Create the Frame: Press F, choose 1440 × 1024 desktop size and name it AppShell/Desktop.

Enable Auto‑layout: Select the frame and press Shift + A to convert it to auto‑layout. Set the direction to Horizontal, so children are arranged left‑to‑right.

Create the Sidebar:

Draw a rectangle (240 px wide, height set to Fill container). Name it Sidebar.

With the rectangle selected, press Shift + A to apply a Vertical auto‑layout.

Add padding of 24 px and set spacing between items to 16 px.

Insert the logo (a 32 px‑high image or placeholder) at the top.

For each navigation item, create a row (40 px high) with a 20 px icon and label. Use auto‑layout horizontally to align icons and labels and apply consistent padding. Turn the row into a component (NavItem) with default, hover, and active variants.

Create the Main Content Area:

Draw another rectangle to the right of the sidebar and set its width to Fill container and height to Fill container.

Apply vertical auto‑layout with padding 48 and spacing 32.

Set the fill color to Surface/App and the frame’s corner radius if needed (e.g., 0 for full‑screen). All pages (progress hub, goals list, etc.) will live inside this area.

.

12. Responsive Rules
Responsive design ensures your interface adapts to smaller viewports:

Sidebar collapses: At narrow widths, change the Sidebar component variant to an icon‑only style (e.g., hide labels and show only icons). You can implement this with a Boolean property to toggle label visibility.

Goal cards: On mobile, stack cards in a single column. Use auto‑layout with Wrap enabled
 to allow items to flow onto new lines.

Level summary: Stack the left and right content vertically when the container width becomes narrow. This can be achieved by creating a responsive variant or by switching auto‑layout orientation based on the parent frame width.

Reward cards: Replace the horizontal scroll container with a vertical list on mobile.

13. Structuring Your Design System Page
To keep your components organised:

Create a new page in Figma called “Design System”.

Add headings (e.g., Buttons, Cards, Typography, Colors, Badges, GoalCard, RewardCard, NavItem, MetricCard).

Place your components under each section. Use auto‑layout to keep them aligned.

Document usage guidelines within Figma notes or use a tool like Notion to write detailed documentation. Make sure to explain the purpose of each component, its variants, and when to use each state.

14. Prototyping
To make your designs interactive:

Select a component (e.g., a button) and switch to the Prototype tab in the right panel.

Drag the blue node from the selected element to the destination frame (e.g., from a GoalCard button to a Goal details page).

Choose Interaction (e.g., On click), Action (Navigate to), and Animation (e.g., Smart animate).

Repeat this for all interactions you want to showcase.
