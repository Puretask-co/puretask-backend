5. Emission helper (server-side)
This gives you a single function to call from any part of your backend (API, cron, webhooks):

// emitJobEvent.ts
import crypto from 'crypto';
import { JobEventName, JobEventPayloadMap, EventEnvelope } from './jobEvents';

const EVENTS_HMAC_SECRET = process.env.EVENTS_HMAC_SECRET!;
const EVENTS_ENDPOINT = process.env.EVENTS_ENDPOINT || 'https://your-n8n-url/events';

export async function emitJobEvent<K extends JobEventName>(
  eventName: K,
  payload: JobEventPayloadMap[K],
  actor: { type: 'client' | 'cleaner' | 'system' | 'admin'; id: string },
  idempotencyKey?: string
) {
  const body: EventEnvelope<K, JobEventPayloadMap[K]> = {
    event_name: eventName,
    event_id: crypto.randomUUID(),
    occurred_at: new Date().toISOString(),
    idempotency_key:
      idempotencyKey ||
      `${eventName}_${(payload as any).job_id || Date.now()}`,
    user_tz: 'America/Los_Angeles', // or pass through from client
    actor,
    payload,
  };

  const rawBody = JSON.stringify(body);

  const hmac = crypto
    .createHmac('sha256', EVENTS_HMAC_SECRET)
    .update(rawBody)
    .digest('hex');

  await fetch(EVENTS_ENDPOINT, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Puretask-Signature': `sha256=${hmac}`,
    },
    body: rawBody,
  });
}

PHASE 2 – Implement the emitJobEvent helper
2.1. Create lib/events/emitJobEvent.ts
// src/lib/events/emitJobEvent.ts
import crypto from 'crypto';
import {
  JobEventName,
  JobEventPayloadMap,
  EventEnvelope,
} from './jobEvents';

const EVENTS_HMAC_SECRET = process.env.EVENTS_HMAC_SECRET!;
const EVENTS_ENDPOINT =
  process.env.EVENTS_ENDPOINT || 'https://YOUR_N8N_URL/webhook/job-events';

type Actor = { type: 'client' | 'cleaner' | 'system' | 'admin'; id: string };

export async function emitJobEvent<K extends JobEventName>(
  eventName: K,
  payload: JobEventPayloadMap[K],
  actor: Actor,
  idempotencyKey?: string
) {
  if (!EVENTS_HMAC_SECRET) {
    console.warn('emitJobEvent: EVENTS_HMAC_SECRET is not set!');
  }

  const body: EventEnvelope<K, JobEventPayloadMap[K]> = {
    event_name: eventName,
    event_id: crypto.randomUUID(),
    occurred_at: new Date().toISOString(),
    idempotency_key:
      idempotencyKey ||
      `${eventName}_${(payload as any).job_id || Date.now()}`,
    user_tz: 'America/Los_Angeles',
    actor,
    payload,
  };

  const rawBody = JSON.stringify(body);

  const hmac = crypto
    .createHmac('sha256', EVENTS_HMAC_SECRET)
    .update(rawBody)
    .digest('hex');

  const res = await fetch(EVENTS_ENDPOINT, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Puretask-Signature': `sha256=${hmac}`,
    },
    body: rawBody,
  });

  if (!res.ok) {
    const text = await res.text().catch(() => '');
    console.error('emitJobEvent failed', res.status, text);
    // up to you: throw error or swallow (I’d log & swallow by default)
  }
}

PHASE 3 – Make sure job_events schema matches
You already have job_events working, but here’s the recommended schema so you can compare:

CREATE TABLE IF NOT EXISTS job_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id UUID NOT NULL REFERENCES jobs(id) ON DELETE CASCADE,
  event_name TEXT NOT NULL, -- e.g. 'cleaner.checked_in'
  occurred_at TIMESTAMPTZ NOT NULL,
  actor_type TEXT NOT NULL, -- 'client' | 'cleaner' | 'system' | 'admin'
  actor_id UUID NOT NULL,
  idempotency_key TEXT NOT NULL,
  payload JSONB NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (idempotency_key)      -- prevents double-insert
);
If yours is already similar, great. If not, adjust via a Neon migration.

PHASE 4 – Configure n8n workflow properly

4.1. Webhook node
Method: POST

Path: /job-events

Respond: 200 JSON { "ok": true }

4.2. (Optional but nice) Signature verification node
Add a Function node after the Webhook:

// Node: Verify Signature
const crypto = require('crypto');

const secret = $env.EVENTS_HMAC_SECRET;
const signatureHeader = $json["headers"]["x-puretask-signature"];

if (!secret || !signatureHeader) {
  throw new Error('Missing secret or signature');
}

const [algo, signature] = signatureHeader.split('=');

const bodyString = JSON.stringify($json["body"]);
const hmac = crypto.createHmac('sha256', secret).update(bodyString).digest('hex');

if (hmac !== signature) {
  throw new Error('Invalid signature');
}

// pass through the body only
return [{ json: $json["body"] }];
Now any request not signed by your backend fails.

(If you don’t want HMAC yet, skip this and just use the body directly.)

4.3. Postgres node (Insert job event)
Configure:

Operation: Insert

Table: job_events

Map fields:

job_id → {{$json["payload"]["job_id"]}}

event_name → {{$json["event_name"]}}

occurred_at → {{$json["occurred_at"]}}

actor_type → {{$json["actor"]["type"]}}

actor_id → {{$json["actor"]["id"]}}

idempotency_key → {{$json["idempotency_key"]}}

import { NextResponse } from 'next/server';
import { emitJobEvent } from '@/lib/events/emitJobEvent';
// import your db client

export async function POST(req: Request) {
  const body = await req.json();

  // 1) Validate input (client_id, schedule, etc.)
  const {
    client_id,
    cleaner_id,
    scheduled_start,
    cleaning_type,
    estimated_hours,
    city,
    state,
    base_rate_cph,
    addon_rate_cph,
    buffer_pct,
  } = body;

  // 2) Insert job into jobs table
  const job = await db
    .insertInto('jobs')
    .values({
      client_id,
      cleaner_id: cleaner_id ?? null,
      scheduled_start,
      cleaning_type,
      estimated_hours,
      city,
      state,
      snapshot_base_rate_cph: base_rate_cph,
      snapshot_addon_cph: addon_rate_cph,
      snapshot_buffer_pct: buffer_pct,
      status: 'scheduled',
    })
    .returningAll()
    .executeTakeFirst();

  // 3) Emit job.created
  await emitJobEvent(
    'job.created',
    {

5.5. Check-out → cleaner.checked_out + job.completed
File: app/api/jobs/[id]/check-out/route.ts

export async function POST(
  req: Request,
  { params }: { params: { id: string } }
) {
  const jobId = params.id;
  const { cleaner_id, gps_lat, gps_lng, distance_meters_from_address } =
    await req.json();

  const job = await db
    .selectFrom('jobs')
    .selectAll()
    .where('id', '=', jobId)
    .executeTakeFirstOrThrow();

  if (!job.checked_in_at) {
    throw new Error('Cannot check-out before check-in');
  }

  const checked_out_at = new Date();
  const total_seconds_worked =
    (checked_out_at.getTime() - new Date(job.checked_in_at).getTime()) / 1000;

  const updated = await db
    .updateTable('jobs')
    .set({
      status: 'completed',
      checked_out_at,
      total_seconds_worked,
    })
    .where('id', '=', jobId)
    .returningAll()
    .executeTakeFirstOrThrow();

  // Event 1: cleaner.checked_out
  await emitJobEvent(
    'cleaner.checked_out',
    {
      job_id: updated.id,
      cleaner_id,
      checked_out_at: checked_out_at.toISOString(),
      gps_lat,
      gps_lng,
      distance_meters_from_address,
      total_seconds_worked,
    },
    { type: 'cleaner', id: cleaner_id }
  );

  // Event 2: job.completed
  await emitJobEvent(
    'job.completed',
    {
      job_id: updated.id,
      client_id: updated.client_id,
      cleaner_id,
      completed_at: checked_out_at.toISOString(),
      total_seconds_worked,
      snapshot_base_rate_cph: updated.snapshot_base_rate_cph,
      snapshot_addon_cph: updated.snapshot_addon_cph,
      snapshot_buffer_pct: updated.snapshot_buffer_pct,
    },
    { type: 'system', id: 'SYSTEM' }
  );

  return Response.json({ job: updated });
}

1️⃣ What are backend API endpoints? (Beginner mode)
Think of your app as having 3 big pieces:

Frontend / apps – what the user taps and sees (Cleaner app, Client app, web)

Backend / API – the “brain” on a server that talks to the database and other services

Database – where all the data lives (Neon/Postgres)

🔌 What’s an API endpoint?
An API endpoint is just a specific URL on your backend that your app calls to do something.

Example (fake URL):

POST https://api.puretask.com/api/jobs/123/check-in

This is like saying:

“Hey PureTask brain, for job 123: the cleaner is checking in now, here are GPS coordinates.”

The backend endpoint:

Receives the request (data coming in)

Checks rules (is this cleaner allowed? Is GPS valid?)

Updates the database (set job as in_progress, save check-in time)

Triggers automations (emit job event to n8n, send notifications)

Sends a response back (e.g., { "ok": true })

🧠 Why do we need endpoints?
Because:

The apps can’t talk directly to your database or n8n safely.

Instead, they talk to the backend through controlled doors (= endpoints).

Endpoints:

Validate data (no garbage, no hacks)

Apply business rules (e.g. can’t check-out before check-in)

Call n8n or Stripe or anything else

Keep secrets safe (API keys, Stripe keys, etc. never live in the app)

So when we say:

“Cleaner taps Check-In”

…what really happens is:

Cleaner app calls a backend endpoint:
POST /api/jobs/:jobId/check-in

Backend:

Updates Neon DB

Emits a job event → sent to n8n

n8n:

Logs event

Sends notifications

Updates analytics, etc.

You don’t put n8n directly in the app; you put endpoints in the backend, and they talk to n8n.

Request body:

  "event_id": "uuid",
  "occurred_at": "2025-11-15T18:21:00Z",

  "actor": {
    "type": "cleaner",
    "id": "cleaner-uuid"
  },
  "payload": {
    "job_id": "job-uuid",
    "cleaner_id": "cleaner-uuid",
    "checked_in_at": "2025-11-15T18:21:00Z",
    "gps_lat": 34.05,
    "gps_lng": -118.25
  }
}
(Optional) Function – “Verify Signature”

Verifies X-Puretask-Signature HMAC using EVENTS_HMAC_SECRET

If invalid → throw error → block bad requests

Postgres – “Insert into job_events”

Operation: Insert

Table: job_events

Map fields:

job_id → {{$json["payload"]["job_id"]}}

event_name → {{$json["event_name"]}}

occurred_at → {{$json["occurred_at"]}}

actor_type → {{$json["actor"]["type"]}}

actor_id → {{$json["actor"]["id"]}}

idempotency_key → {{$json["idempotency_key"]}}

payload → {{$json["payload"]}}

Turn on “Continue on Fail” = false so duplicates (same idempotency_key) just error and stop.

Switch – “Route by event_name”

Switch on: {{$json["event_name"]}}

Cases:

job.cleaner_on_my_way → On My Way workflow
