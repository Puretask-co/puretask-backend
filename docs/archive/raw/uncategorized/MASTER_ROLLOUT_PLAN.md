### **4. DEVELOPMENT TIPS**

**Mobile-First:**
- Design for iPhone 12/13 size first (390px wide)
- 70% of bookings will be on mobile
- Test on real devices

**Component Reuse:**
- Build once, use everywhere
- Your Button component should be used 100+ times
- Same with Card, Modal, Input

**Performance:**
- Lazy load pages: `const Page = lazy(() => import('./Page'))`
- Optimize images: Use Next.js Image component
- Cache API responses: Use React Query
- Monitor: Set up Vercel Analytics

**Testing:**
- Test payment flow REPEATEDLY
- Test on real phones
- Get 5 beta users to test everything
- Fix bugs before adding features

---

### **5. COMMON MISTAKES TO AVOID**

❌ **Don't:**
- Build all 50 pages at once
- Perfect one page before moving on
- Build features users haven't requested
- Neglect mobile responsive design
- Skip error handling
- Forget loading states
- Ignore accessibility

✅ **Do:**
- Build page by page
- Test each page
- Get feedback early
- Mobile-first design
- Handle all edge cases
- Show loading spinners
- Use semantic HTML
