2. BACKEND FEATURES (APIS & SERVICES)
Below are the main features Cursor should implement as TypeScript services + routes (exact framework assumed: Express-like).

2.1 Subscription Cleanings
Core behavior:

Client can create a subscription for a property: weekly/bi-weekly/monthly.

System auto-creates upcoming jobs & holds credits when subscription date arrives.

Client can pause/cancel, change cadence, change preferred cleaner.

Endpoints (example):

POST /api/subscriptions
GET  /api/subscriptions
GET  /api/subscriptions/:id
PATCH /api/subscriptions/:id
POST /api/subscriptions/:id/pause
POST /api/subscriptions/:id/resume
POST /api/subscriptions/:id/cancel
Service responsibilities:

SubscriptionService.create:

Validate property belongs to client.

Compute next_run_at based on cadence & preferred day/time + city timezone.

Insert row into subscriptions.

SubscriptionScheduler.runDueSubscriptions (cron/n8n-triggered):

Find active subscriptions with next_run_at <= now.

For each:

Create a job record using snapshot pricing & property.

Hold credits in escrow (same existing logic).

Emit event subscription.job_created.

Advance next_run_at by cadence.

2.2 One-Tap Rebook
Goal: From a completed job, client taps “Rebook” → new job created with same cleaner/property/hours.

Endpoint:

POST /api/jobs/:id/rebook
Logic:

Validate original job belongs to client & status is approved or completed.

Use same property, cleaner (if still active), cleaning type, base hours.

Suggest next available slot based on cleaner availability + city rules.

Create job + escrow hold.

2.3 Cleaning Score Per Home
Concept:

Track how recently each property had a basic/deep/move-out cleaning.

Compute a 0–100 “cleaning score” based on recency.

Implementation:

When a job is completed/approved:

Update properties.last_basic_at/last_deep_at/last_moveout_at.

Recompute cleaning_score, e.g.:

score = 100 - (days_since_last_clean * weight)
clamp between 0 and 100
Expose in:

GET /api/properties/:id
GET /api/properties (list with scores)
Add suggestions endpoint:

GET /api/properties/:id/suggestions
Returns messages like:
